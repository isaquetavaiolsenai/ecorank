# Fase 2: Interface & UX de Diagnóstico

## Objetivo da Fase
Criar uma jornada de integração fluida e visualmente impactante para o usuário descobrir seu perfil ecológico.

## Tasks

### Task 2.1: Sistema de Navegação Modular
* **O que mudará:** Refatoração do `ViewState` para suportar transições animadas entre categorias.
* **Necessita banco de dados (SQL):** Não.
* **Conexão com o Objetivo:** Fluidez na experiência.

### Task 2.2: Componente de Progresso Dinâmico
* **O que mudará:** Barra de progresso circular que se preenche conforme o usuário responde.
* **Necessita banco de dados (SQL):** Não.
* **Conexão com o Objetivo:** Feedback visual de avanço.

### Task 2.3: Injeção de Dicas em Tempo Real via AI
* **O que mudará:** Pequenos popups de curiosidades baseados nas respostas atuais.
* **Necessita banco de dados (SQL):** Não.
* **Conexão com o Objetivo:** Educação imediata.

### Task 2.4: Persistência de Respostas Parciais
* **O que mudará:** Salvar no LocalStorage para que o usuário não perca o progresso se fechar o app.
* **Necessita banco de dados (SQL):** Não.
* **Conexão com o Objetivo:** Redução de fricção.

### Task 2.5: Design de Ícones Personalizados "Eco-Minimal"
* **O que mudará:** Adição de ícones da lucide-react estilizados via CSS para categorias.
* **Necessita banco de dados (SQL):** Não.
* **Conexão com o Objetivo:** Consistência estética.

### Task 2.6: Lógica de Ponderação de Pontos
* **O que mudará:** Algoritmo que diferencia perguntas críticas de perguntas informativas.
* **Necessita banco de dados (SQL):** Não.
* **Conexão com o Objetivo:** Precisão no diagnóstico.

### Task 2.7: Tradução de Termos Técnicos
* **O que mudará:** Substituir "ESG" ou "Carbon Footprint" por termos mais palatáveis.
* **Necessita banco de dados (SQL):** Não.
* **Conexão com o Objetivo:** Acessibilidade cognitiva.

### Task 2.8: Mockup de Resultado "Impact Card"
* **O que mudará:** Tela de resumo com estatísticas e gráficos de pizza.
* **Necessita banco de dados (SQL):** Não.
* **Conexão com o Objetivo:** Valor percebido.

### Task 2.9: Sistema de Swipe para Questões
* **O que mudará:** Interface inspirada em apps de swipe para tornar a pesquisa lúdica.
* **Necessita banco de dados (SQL):** Não.
* **Conexão com o Objetivo:** Engajamento.

### Task 2.10: Validação de Campos Obrigatórios
* **O que mudará:** Bloqueio do botão "Próximo" caso perguntas essenciais faltem.
* **Necessita banco de dados (SQL):** Não.
* **Conexão com o Objetivo:** Qualidade dos dados.
